Git Basics

  • Understanding Version Control Systems (VCS)
    A Version Control System (VCS) tracks changes to files over time, enabling multiple people to collaborate on projects and allowing you to revert to previous versions if needed. Git is one of the most popular VCSs, known for its distributed nature and efficiency in managing code changes.

  • Initializing a Git Repository
    To start tracking a project with Git, you need to initialize a repository.

    Example:

    # Navigate to your project directory
cd my-project

# Initialize a Git repository
git init

# This creates a hidden .git directory in your project folder, where Git stores the version history.

  • Basic Git Commands

    • git add: Stages changes (files or directories) to be committed.

      # Stage a specific file
git add filename.py

# Stage all changes in the directory
git add .

    • git commit: Records the staged changes in the repository’s history.

      # Commit with a message describing the changes
git commit -m "Initial commit"

    • git push: Uploads your local repository changes to a remote repository (like GitHub).

      # Push changes to the remote repository (usually 'origin') on the 'main' branch
git push origin main

    • git pull: Fetches and integrates changes from a remote repository into your local repository.

      # Pull the latest changes from the remote repository
git pull origin main

    • git merge: Combines changes from different branches into one.

      # Merge the 'feature' branch into the current branch
git merge feature

Working with GitHub

  • Creating and Cloning Repositories
    GitHub is a cloud-based platform that hosts Git repositories and enables collaboration on open-source and private projects.

    • Creating a Repository on GitHub:

      1. Log in to your GitHub account.
      2. Click on “New” to create a new repository.
      3. Name your repository and optionally add a description.
      4. Choose between public or private visibility.
      5. Click “Create repository”.

    • Cloning a Repository:

      # Clone a repository to your local machine
git clone https://github.com/username/repository.git

# This will create a directory named 'repository' with the project files.

  • Pull Requests and Code Reviews
    A pull request (PR) is a method of submitting contributions to a project. Other team members can review the changes before merging them into the main codebase.

    Creating a Pull Request:

    1. Push your changes to a branch on GitHub.
    2. Navigate to the repository on GitHub.
    3. Click the “Compare & pull request” button.
    4. Review your changes and add comments if necessary.
    5. Click “Create pull request”.

    Code Reviews:

    • Code reviews involve team members examining the PR to ensure quality, spot bugs, and suggest improvements before the changes are merged.

  • Managing Branches and Resolving Conflicts
    Branches in Git allow you to work on different features or fixes independently of the main codebase.

    • Creating a Branch:

      # Create and switch to a new branch named 'feature-branch'
git checkout -b feature-branch

    • Switching Between Branches:

      # Switch back to the 'main' branch
git checkout main

    • Resolving Conflicts:
      Conflicts occur when two branches have changes to the same part of a file. Git will stop the merge and mark the conflict in the file.

      Steps to Resolve a Conflict:

      1. Open the file with conflicts and decide which changes to keep.
      2. Edit the file to resolve the conflicts.
      3. Stage the resolved file with git add.
      4. Commit the changes with git commit.

      Example:

      # After resolving the conflict
git add conflicted_file.py
git commit -m "Resolved merge conflict in conflicted_file.py"
